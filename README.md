## 🇮🇹 Italiano

Questo repo contiene le card mantenute direttamente dal team Oikos. È preconfigurato come sorgente nello Store della dashboard:

> **Store → Comunità → JAVA** → `https://github.com/Bobsilvio/oikos-cards`

Ogni release pubblica i bundle ESM pronti all'uso (CI in `.github/workflows/publish.yml`).

### ✋ Vuoi creare una tua card?

**Non clonare questo repo.** Usa lo starter dedicato:

> 👉 **[github.com/Bobsilvio/oikos-card-starter](https://github.com/Bobsilvio/oikos-card-starter)**

Lo starter contiene:
- template `cards/my-card/` pronto da modificare
- 3 esempi commentati (hello world, sensor, light toggle)
- documentazione passo-passo (`docs/`)
- skill per agenti AI (`SKILL.md`)
- comando `npm run build:my` → ZIP installabile

### Struttura del repo

```
cards/
  <card-id>/
    manifest.json   ← metadati della card
    src/Card.jsx    ← sorgente
    dist/           ← bundle prodotto da build-card.mjs
    preview.png     ← opzionale, anteprima nello Store (3:2, max 600×400)
manifest.json       ← aggregato (generato da generate-manifest.mjs)
tools/              ← vite plugin SDK, build, publish, manifest
```

### Comandi

```bash
npm ci                              # installa deps (Vite, plugin React)
npm run build cards/<id>            # build singola card
npm run build:all                   # build tutte + manifest aggregato
npm run manifest                    # solo rigenerazione manifest aggregato
./tools/publish-cards.sh --tag v1.0 # build + manifest + tag git
```

### Aggiungere una card al catalogo ufficiale

Una card finisce in questo repo solo se è mantenuta dal team Oikos:

1. Sviluppa la card nello starter (workflow consigliato)
2. Apri PR copiando `cards/<id>/` qui dentro
3. La CI buildera e pubblicherà al prossimo tag `v*`

### Policy

- `id` card: `^[a-z0-9][a-z0-9_-]{0,63}$`, opzionalmente namespaced `owner/card`
- `sdkVersion`: semver range, es. `^1.0.0`
- `configVersion`: incrementare in breaking change di schema config, con `migrate(old, fromVersion)` nella card
- Niente fetch verso host esterni non dichiarati, niente accesso a `auth.access_token`, niente CSS globale

### Skill per agenti AI

Lo skill ufficiale è in `SKILL.md`. Puoi linkarlo a Claude / agenti per abilitare l'assistenza alla creazione di card.

## 🇬🇧 English

This repo contains cards maintained directly by the Oikos team. It is pre-configured as a source in the dashboard Store:

> **Store → Community → JAVA** → `https://github.com/Bobsilvio/oikos-cards`

Each release publishes ready-to-use ESM bundles (CI via `.github/workflows/publish.yml`).

### ✋ Want to build your own card?

**Don't clone this repo.** Use the dedicated starter:

> 👉 **[github.com/Bobsilvio/oikos-card-starter](https://github.com/Bobsilvio/oikos-card-starter)**

The starter includes:
- `cards/my-card/` template ready to edit
- 3 commented examples (hello world, sensor, light toggle)
- step-by-step documentation (`docs/`)
- AI agent skill (`SKILL.md`)
- `npm run build:my` command → installable ZIP

### Repo structure

```
cards/
  <card-id>/
    manifest.json   ← card metadata
    src/Card.jsx    ← source
    dist/           ← bundle produced by build-card.mjs
    preview.png     ← optional, Store preview image (3:2, max 600×400)
manifest.json       ← aggregated manifest (generated by generate-manifest.mjs)
tools/              ← SDK vite plugin, build, publish, manifest
```

### Commands

```bash
npm ci                              # install deps (Vite, React plugin)
npm run build cards/<id>            # build single card
npm run build:all                   # build all + aggregated manifest
npm run manifest                    # regenerate aggregated manifest only
./tools/publish-cards.sh --tag v1.0 # build + manifest + git tag
```

### Adding a card to the official catalog

A card lands in this repo only if it is maintained by the Oikos team:

1. Develop the card in the starter (recommended workflow)
2. Open a PR copying `cards/<id>/` here
3. CI will build and publish on the next `v*` tag

### Policy

- Card `id`: `^[a-z0-9][a-z0-9_-]{0,63}$`, optionally namespaced `owner/card`
- `sdkVersion`: semver range, e.g. `^1.0.0`
- `configVersion`: increment on breaking config schema changes, with `migrate(old, fromVersion)` in the card
- No fetches to undeclared external hosts, no access to `auth.access_token`, no global CSS

### AI agent skill

The official skill is in `SKILL.md`. Link it to Claude / agents to enable card creation assistance.
